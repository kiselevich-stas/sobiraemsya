# Настройка Supabase для «Собираемся?»

## 1. Создай проект

Создай новый проект в Supabase Dashboard.

## 2. Создай таблицу

Открой **SQL Editor** и выполни файл:

```sql
supabase/schema.sql
```

Таблица хранит событие целиком в `payload jsonb`. Это быстрее для MVP: не нужно сразу проектировать сложную схему из нескольких таблиц.

## 3. Добавь переменные окружения

Скопируй `.env.example` в `.env.local`:

```bash
cp .env.example .env.local
```

Заполни:

```env
VITE_SUPABASE_URL=https://xxxx.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=твой_publishable_key
```

Если в интерфейсе Supabase у тебя пока показан `anon public key`, можно использовать:

```env
VITE_SUPABASE_ANON_KEY=твой_anon_key
```

## 4. Запусти приложение

```bash
npm install
npm run dev
```

Если переменные не заполнены, приложение автоматически работает через localStorage.

## 5. Как работает синхронизация

- создание события сохраняет его в localStorage и Supabase;
- открытие ссылки `/event/:id` загружает событие из Supabase;
- отметки участников, задач и платежей обновляют JSON-событие в Supabase;
- Realtime подписка подтягивает изменения на открытых устройствах.

## Ограничение MVP

Сейчас включён режим «любой, у кого есть ссылка, может редактировать событие». Для бытовых сборов в групповом чате это нормально для MVP, но для публичного запуска лучше добавить `edit_token`, Telegram initData-проверку или полноценную авторизацию.
